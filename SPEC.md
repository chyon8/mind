# SPEC.md — 프로젝트 기획서 (개발 착수 전 1회 정독)

## 1. 이 앱이 무엇인가

북마크앱도, 투두앱도, 생산성 앱도 아니다.
**사용자(단 한 명)의 기억과 뇌를 흉내내는 아이디어 파편 저장소**다.

사용자의 행동 패턴: 아이디어 파편이 수시로 떨어진다. 놓치기 싫어서 던져두지만
다시 잘 보지 않는다. 평생 정리를 하지 않는 사람이며, 정리를 요구하는 앱은 반드시 죽는다.
문제는 저장도 검색도 아니다 — **전부 남기니까 전부 안 보이는 것.**
파편이 같은 무게로 평평하게 쌓여 벽이 되는 것이 유일한 병목이다.

해결 원리 — 기억처럼 작동시킨다:

> **파편은 던진 순간 선명하고, 시간이 지나면 흐려지고, 건드리면 다시 선명해진다.**

잊히는 것은 정상이고 건강한 것이다. 안 보는 행위 자체가 판정이다.
사용자가 아무것도 정리하지 않아도 시스템은 완전히 작동해야 한다.

## 2. 절대 원칙 (구현 결정과 충돌 시 이쪽이 이긴다)

1. **입력 마찰 0** — 저장 시점에 어떤 결정도 요구하지 않는다.
   타입 선택·프로젝트 선택·태그 전부 금지. 자동이거나, 나중이거나.
2. **생산성 문법 배제** — 완료 체크, 달성률, 스트릭, 마감일, 알림 없음.
   태스크 도구가 아니라 자기 관찰 도구다.
3. **폴더/태그 없음** — 시간축과 선명도가 구조를 대신한다.
4. **프로젝트는 폴더가 아니라 필터(렌즈)다** — 같은 파편이 전체 타임라인에도,
   프로젝트 뷰에도 보인다.
5. **사용자는 한 명** — 멀티유저 고려, 온보딩, 일반화 금지. 과잉 추상화 금지.
   엔터프라이즈 패턴(DI 컨테이너, 과도한 모듈 분리) 도입 전에 "1인용 앱에 필요한가"를 물을 것.

## 3. 스택

- **iOS 앱**: Expo (React Native + TypeScript)
- **서버**: Supabase (Postgres + Auth + Storage + REST) — 서버 코드 직접 작성 없음, 무료 티어
- **웹 입력 페이지**: 순수 HTML/JS 단일 페이지, Supabase JS로 직접 insert
  (회사 데스크탑에서 던지기용. 뷰 기능 없음 — 입력창 + 던지기 버튼이 전부)
- **공유 저장 (expo-share-intent)**: 필수. 이 앱의 심장. 사파리/유튜브/사진 어디서든 2탭 저장
  (공유 → Mind 선택 → 앱 열리며 자동 저장). 이게 안 되면 앱 전체가 무의미하므로 최우선 구현 대상

## 4. 데이터 모델

**Fragment**
| 필드 | 설명 |
|---|---|
| id, created_at | 기본 |
| content | 원문 |
| type | text / link / image / quote — **자동 판별** |
| link_title, link_thumbnail_url | link일 때 비동기 수집 |
| image_path | image일 때 (Supabase Storage) |
| last_touched_at | 선명도 계산의 유일한 입력값 |
| tier | normal / important / pinned |
| project_id | nullable. null = inbox. 사용자가 나중에 붙임 |
| archived | 바닥 여부 |

타입 자동 판별: URL 패턴 → link, 이미지 파일 → image, 인용부호/출처 형태 → quote, 나머지 → text.

**선명도는 저장하지 않는다.** last_touched_at + tier로 뷰 레이어에서 계산.
데이터는 사실만, 해석은 화면에서.

**Project**
| 필드 | 설명 |
|---|---|
| id, name | 기본 |
| status | before / active / paused / done |

이 이상 필드 추가 금지. 태스크·마감일·진척률은 이 앱의 적이다.
프로젝트는 "지금 내 삶에서 뭐가 살아있나"를 보는 지도일 뿐, 관리 도구가 아니다.

## 5. 선명도 규칙 (확정값)

| tier | 감쇠 시작 | 바닥(투명도 25%) 도달 |
|---|---|---|
| normal | 7일 | 30일 |
| important | 30일 | 90일 |
| pinned | 감쇠 없음 | — |

- 파편을 열면 last_touched_at 갱신 → 100% 복귀
- 바닥 도달 후에도 리스트에 남는다(25% 고정). 삭제는 수동만
- 감쇠 곡선은 선형으로 시작

## 6. 화면 (MVP)

1. **타임라인 피드 (기본)** — 카드 단위, 최신 위, 날짜 구분선.
   선명도를 카드 opacity로 시각화 — 스크롤 내려 과거로 갈수록 연해지는 지층감이 이 앱의 정체성.
   상단에 필터 칩 (All + 진행중 프로젝트만). Inbox·무덤·전체 프로젝트 목록(상태별)은
   좌측 사이드바(엣지 스와이프로 열고 닫음)에서 접근. 파편은 탭→상세, 우→좌 스와이프→수정/삭제
2. **어젠다 뷰** — 날짜가 섹션 헤더(큰 숫자+요일), 아래 파편이 한 줄 행으로 밀도 높게.
   훑기용. opacity 감쇠 동일 적용. 피드와 상단 토글로 전환
3. **빠른 입력** — 붙여넣기/타이핑 → 타입 자동 인식 배지 → 던지기.
   프로젝트 선택은 접힌 옵션(기본 Inbox)
4. **파편 상세** — 원문 전체. tier 3단 토글, 프로젝트 붙이기, 삭제. 열리는 순간 touch 갱신
5. **공유 저장** — 공유 시트에서 Mind 선택 → 앱 열리며 자동 저장 + 토스트. 프로젝트 선택 넣지 말 것
6. **웹 입력 페이지**

세부 스타일은 design.md를 따른다.
