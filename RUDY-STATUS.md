## 구현 중 내린 결정 (RUDY-BUILD 대비 변경/확정)

- **링크 임베딩 = 제목+og:description** (요약 아님). 유튜브는 og:description이 홍보문구로 오염돼
  페이지의 `ytInitialPlayerResponse.shortDescription`을 직접 파싱 (linkMeta.ts + 스크립트 동일 로직).
- **다이제스트(LLM 요약·의도) 보류.** 비용이 아니라 — (1) 보여줄 표면 없음, (2) 검색품질은
  설명글 임베딩으로 이미 해결, (3) 의도 해석은 lazy가 §2-1 정합. → 채팅 만들 때 lazy 계산.
- **검색 폴백**: 임베딩/RPC 실패 시 키워드 검색으로 자동 폴백(+console.warn).
- **타입 필터 = 서버(RPC)에서** (클라 후필터 아님).
- 스크립트는 `.mjs` (TS 러너 미설치). embedText/해시는 embed 함수와 반드시 동일.
- **충돌: seed 평균 임베딩 폐기 → seed별 개별 비교 후 최댓값** (RUDY-BUILD B-1에서 변경).
  이유 (1) 무관한 걸 여러 개 던진 날 평균은 노이즈 중심이 되어 억지 충돌을 만든다,
  (2) "왜 지금"(B-3)이 어느 씨앗과 부딪혔는지를 요구하는데 평균은 그 정보를 지운다.
- **충돌 게이트와 순위 분리** (RUDY-BUILD B-2에서 변경). B-2의 `sim×faded×weight ≥ 0.35`는
  weight가 위로 열려 있어(1 + touch_count×2 + …) **자주 만진 파편이 무관해도 통과**하고 평범한
  파편은 관련 있어도 못 통과한다 — 억지 충돌 방지 장치가 정반대로 작동. → 게이트는 `sim`만,
  통과한 것 중 순위만 `sim × faded × weight`.
- **선명도는 SQL로 옮기지 않는다.** 감쇠 법칙의 단일 원천은 vividness.ts. RPC는 "아직 흐려졌을
  리 없는 것"(7일 미만)만 보수적으로 걷어낸다. 진단용 `min_age_days=0` 파라미터로 필터를 끌 수 있음.
- **충돌 임계값 0.35 → 0.42** (실측, `check-collision.mjs --all`). 실제 코퍼스 41개 후보로 보니
  0.35는 깨끗한 경계가 아니었음 — teenage engineering 클러스터는 0.52~0.73으로 확실했지만
  0.30~0.34대에 무관한 것과 관련 있는 것이 뒤섞여 있었다(임베딩 이방성). 억지 충돌보다 놓치는
  쪽이 안전하다는 §2-8에 따라 올림. **진짜 튜닝은 원장(§10-4) 게이트 판정 로그로 — 이 값은 잠정.**

## §10-4 원장 — 구현 범위 (의도적으로 좁힘)

**만든 것:** `rudy.utterances`(원장) + `rudy.gate_log` + 30일 되살리기 쿨다운 + acted/dismissed 캡처.

**안 만든 것 — §6-4의 5개 게이트를 추상 프레임워크로 만들지 않았다.** 지금 말하는 주체가
충돌 회상 하나뿐이라 범용 파이프라인은 speculative abstraction(CLAUDE.md §2)이다. 실제로 쓰이는
게이트만 인라인으로 있다: **유사도 게이트**, **쿨다운 게이트**. 화법·확신 게이트는 생성 발화가
없어서 적용 대상 자체가 없고, 예산은 하루 1개 저장(AsyncStorage day key)으로 이미 강제된다.
→ **두 번째 화자(채팅)가 생길 때 공통 모양이 드러나면 그때 추출한다.**

설계 판단 기록:
- `item_ids`에 **씨앗은 넣지 않는다.** "왜 지금"은 렌더 시점 계산·무저장이고 연결 저장은
  Mind SPEC §7 금지. 씨앗은 기기 로컬(AsyncStorage)에 하루치만 남는다.
- **랜덤 픽은 원장에 안 들어간다.** 그건 루디의 발화가 아니라 우연의 자리(§4-F2)다.
  따라서 랜덤으로 뜬 파편에 기억하기/흘려보내기를 해도 루디 성적에 반영되지 않는다 — 의도된 것.
- `ignored`는 저장하지 않는다. "응답 없음 + 24h 경과"로 집계 시점 계산 (노출은 신호가 아니다 §2-3).
- 원장 기록 실패가 회상을 죽이지 않는다 (try/catch + warn).
